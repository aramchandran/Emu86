PyGotham Talk Outline

  • The problem: in a course not focused on assembler, it is difficult to introduce "a little" assembler.
    • We don't want students to have to download and install something if we can avoid it.
    • A web-based solution is the answer.
  • Python makes coding an interpreter easy.
    • Discussion of what Python features we used.
  • Django makes bringing up a web app with a database behind it easy.
    • Discussion of what Django features we used.
  • DevOps design considerations:
    • Service oriented: Keep the interpreter and the interface completely separate. (In fact, we can run from within iPython as well.)
    • Automate testing.
    • Automate documentation.
    • Automate deployment.
  • Success in using the tool.
